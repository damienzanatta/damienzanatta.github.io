Projet 3

Configuration DHCPv4

Présentation

Dans le cadre de ma formation, j’ai réalisé un TP Cisco Packet Tracer portant sur la configuration du service DHCPv4.

L’objectif était d’automatiser l’attribution des paramètres IPv4 aux clients d’un réseau, en configurant un routeur Cisco comme serveur DHCP, des relais DHCP et une interface cliente DHCP.

Objectifs

  • Configurer un routeur comme serveur DHCPv4.
  • Exclure les adresses IPv4 réservées.
  • Créer des pools DHCP adaptés aux réseaux locaux.
  • Configurer le relais DHCP sur les routeurs concernés.
  • Configurer une interface routeur en client DHCP.
  • Vérifier l’attribution des adresses et la connectivité.

Environnement technique

  • Logiciel : Cisco Packet Tracer
  • Équipements : routeurs Cisco, commutateurs Cisco, postes clients et serveur DNS
  • Interface : CLI Cisco IOS
  • Technologies : DHCPv4, relais DHCP, adressage IPv4 et routage

Topologie réseau et table d'adressage

La topologie est composée de plusieurs routeurs interconnectés, de réseaux locaux clients et d’un serveur DNS.

La table d’adressage permet d’identifier les réseaux concernés, les passerelles par défaut et les adresses devant être distribuées automatiquement aux clients.

Étape 1 – Configuration du serveur DHCPv4 sur R2

Je suis d’abord entré dans le routeur R2 afin de le configurer comme serveur DHCPv4 pour plusieurs réseaux locaux.

J’ai exclu les premières adresses des réseaux concernés afin d’éviter qu’elles soient attribuées automatiquement à des clients.

Cette étape permet de réserver les adresses nécessaires aux interfaces de routeurs, aux serveurs ou aux équipements d’administration.

Étape 2 – Création des pools DHCPv4

J’ai créé les pools DHCP destinés aux réseaux locaux qui doivent recevoir une configuration IPv4 automatiquement.

Pour chaque pool, j’ai renseigné le réseau concerné, la passerelle par défaut et le serveur DNS.

Cette configuration permet aux postes clients de recevoir automatiquement une adresse IP, un masque, une passerelle et une adresse de serveur DNS.

Étape 3 – Configuration du relais DHCP

J’ai configuré le relais DHCP sur les interfaces LAN des routeurs qui ne possèdent pas directement le serveur DHCP.

J’ai utilisé la commande ip helper-address afin de rediriger les requêtes DHCP des clients vers le routeur R2.

Cette configuration permet à des clients situés sur un autre réseau de recevoir une adresse IP depuis un serveur DHCP distant.

Étape 4 – Configuration d’une interface en client DHCP

J’ai configuré une interface du routeur R2 afin qu’elle obtienne automatiquement une adresse IPv4 par DHCP.

J’ai utilisé la commande ip address dhcp, puis j’ai activé l’interface avec la commande no shutdown.

Cette étape permet de simuler le comportement d’un routeur recevant une configuration IP depuis un réseau fournisseur.

Étape 5 – Vérifications DHCPv4

J’ai vérifié que les clients recevaient bien leurs informations réseau automatiquement.

J’ai également consulté les informations DHCP afin de contrôler les adresses attribuées et les liaisons créées par le serveur.

Ces vérifications permettent de confirmer que le service DHCPv4 fonctionne correctement sur les réseaux concernés.

Étape 6 – Tests de connectivité

J’ai réalisé des tests de connectivité afin de vérifier que les clients pouvaient communiquer avec les équipements du réseau.

Les tests ont permis de confirmer que les adresses attribuées par DHCP étaient exploitables et que les passerelles étaient correctement configurées.

Cette dernière vérification valide la configuration DHCPv4 et le bon fonctionnement de l’adressage automatique.

Résultats obtenus

  • Routeur R2 configuré comme serveur DHCPv4.
  • Adresses IPv4 réservées exclues des pools DHCP.
  • Pools DHCP créés pour les réseaux locaux.
  • Passerelles et serveur DNS renseignés dans les pools.
  • Relais DHCP configuré sur les routeurs concernés.
  • Interface de R2 configurée en client DHCP.
  • Attribution automatique des adresses IPv4 vérifiée.
  • Connectivité réseau validée.

Bilan personnel

Ce TP m’a permis de mieux comprendre le fonctionnement du service DHCPv4 et son rôle dans l’automatisation de l’adressage réseau.

J’ai appris à créer des pools DHCP, à exclure des adresses réservées, à configurer un relais DHCP et à vérifier les adresses attribuées aux clients.

Cette réalisation est importante car DHCP est un service essentiel pour simplifier l’administration des postes clients dans une infrastructure réseau.
